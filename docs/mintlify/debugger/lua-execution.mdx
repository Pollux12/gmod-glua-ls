## Overview

While the debugger is connected (not necessarily paused), you can execute Lua code on the server from VS Code in several ways.

All options below (apart from refresh) require the execution realm to be selected. This determines what realm it will run on.

---

## Realm targeting

By default, code runs on the **server** realm. You can change the selected realm via the extension via either:

1. Command Palette → **GLua: Set Realm**
2. Realm selector on bottom status bar

When targeting `client`, code runs on all connected players. This works by sending a net message to all clients with the code to execute. This code can be found within the `debug.lua` file in autorun - this is automatically generated on install.

---

## Run selected Lua

1. Select any Lua code in the editor
2. Right-click → **GLua: Run Selected**

The selected code is sent to the specified realm and executed. Output appears in the **Debug Console**.

---

## Run current file

<Info>
This runs the current file content as saved within the workspace. The server will be sent this file path to then execute.
</Info>

Send the entire current file's content for execution:

1. Open the Command Palette (`Ctrl+Shift+P`)
2. Run **GLua: Run Current File**

---

## Run a server command

Execute a SRCDS console command:

1. Open the Command Palette
2. Run **GLua: Run Command**
3. Type the command (e.g., `lua_run print("Hello")`)

Output is shown in the Debug Console.

---

## Refresh File (Lua Refresh / Hot Reload)

Refresh a file on the server without restarting. This is useful if you have lua refresh disabled.

1. Make your changes in the editor and save
2. Right-click in the editor → **GLua: Refresh File on Server**

This is the same as triggering lua refresh on the specified file.

---

## Debug Console

The **Debug Console** (`Ctrl+Shift+Y`) is the most direct way to interact. Any input is evaluated as a Lua expression on the server (or the targeted realm):

```lua
> print(game.GetMap())
gm_flatgrass

> for k, v in pairs(ents.GetAll()) do print(v:GetClass()) end
prop_physics
prop_physics
player
-- ...
```

This is equivalent to using `lua_run` in the server console, but with output returned to VS Code.
