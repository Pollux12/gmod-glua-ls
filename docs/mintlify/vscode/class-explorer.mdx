## Overview

The Class Explorer is a tree view in the VS Code Activity Bar (the icon strip on the left). It provides two main sections:

- **Classes** — Scripted class definitions found in your workspace (entities, weapons, effects, custom classes)
- **Resources** — Materials, models, and sounds referenced in your project

Click any class or file to jump directly to its definition.

## Classes section

The Classes section groups your scripted classes by type:

| Group | Detected from |
|---|---|
| **Entities** | Files under `lua/entities/` or custom `path` config |
| **Weapons** | Files under `lua/weapons/` |
| **Effects** | Files under `lua/effects/` |
| **Scripted Tools** | Files under `lua/weapons/gmod_tool/stools/` |
| **Gamemodes/Plugins** | Files under configured plugin paths |

Each group shows the detected class name alongside its class global (`ENT`, `SWEP`, etc.). Expand a class to navigate to its definition.

### Creating new classes

When a scripted class definition has scaffold files configured, a **New** button appears next to the class group. Clicking it:

1. Prompts you for the new class name
2. Generates all template files for that class in the correct location
3. Opens the main file immediately

See [Scaffolding](/vscode/scaffolding) for details on configuring templates.

---

## Resources section

The Resources section shows workspace assets organized by type:

| Category | Contents |
|---|---|
| **Materials** | `.vmt` and `materials/` references found in Lua files |
| **Models** | `models/` path references |
| **Sounds** | `sounds/` and `sound/` references |

Click any resource to open the file it was referenced from.

---

## Filtering and search

Use the search box at the top of the Class Explorer to filter classes by name. The filter applies to the class name, its global (`ENT`, `SWEP`, etc.), and file path.

## Custom class groups

You can define custom class groups for frameworks that use their own class patterns. See [Scripted class definitions](/vscode/settings-ui#scripted-class-definitions) and the [`gmod.scriptedClassScopes`](/configuration/gmod#scriptedclassscopes) configuration section for details.

Example: adding a `Controllers` group for a custom framework:

```json .gluarc.json
{
  "gmod": {
    "scriptedClassScopes": {
      "include": [
        {
          "id": "my-controllers",
          "label": "Controllers",
          "classGlobal": "CONTROLLER",
          "path": ["myframework", "controllers"],
          "include": ["lua/myframework/controllers/**"],
          "rootDir": "lua/myframework/controllers",
          "icon": "symbol-class"
        }
      ]
    }
  }
}
```
