title: "@async"
description: Mark a function as coroutine-based or asynchronous.
---

## Overview

`@async` marks a function as asynchronous — typically meaning it uses `coroutine.yield` internally and should be called within a running coroutine. It serves as documentation and enables future async-aware diagnostics.

---

## Syntax

```lua
---@async
```

---

## Usage

```lua
---@async
---@param url string The URL to fetch
---@return string response The response body
function fetch(url)
    -- internally yields waiting for I/O
    local response = http.Fetch(url)
    coroutine.yield()
    return response
end
```

---

## What it does

- Marks the function in hover documentation as async
- Allows tools and linters to warn when an `@async` function is called outside a coroutine context (if that diagnostic is enabled)
- Has no runtime effect

---

## Common GMod patterns

GLuaLS supports async-style workflows using coroutines and callback-based APIs. GMod does not include a built-in `await()` function, but many addons use helper wrappers to mimic this behavior.

```lua
---@async
function PLUGIN:FetchPlayerData(steamId)
    local response
    http.Fetch("https://api.example.com/player/" .. steamId, function(body)
        response = body
    end)

    coroutine.yield() -- wait for the callback to set `response`
    return response
end
```

`await-in-sync` support is available in GLuaLS for equivalent coroutine helper patterns, so this model can be checked by the server-side analysis.
