## Overview

`@version` marks a definition as only available on certain Lua versions.

GLuaLS uses this together with `runtime.version` when it decides whether a field or function should be visible.

Garry's Mod projects usually target LuaJIT, so this annotation is mostly useful in shared annotation files and standard-library docs.

---

## Syntax

```lua
---@version >5.1, JIT
```

Operators: `>`, `<`.

- `> 5.1` means Lua 5.1 and newer
- `< 5.4` means Lua 5.4 and older
- Versions: `5.1`, `5.2`, `5.3`, `5.4`, `JIT`

---

## Usage

```lua
---@version >5.2
function string.pack(fmt, ...) end

-- If your workspace is set to LuaJIT, this definition is treated as unavailable.
```

---

## Compatibility

Set the workspace version with `runtime.version`. In a Garry's Mod project, that usually means LuaJIT.
