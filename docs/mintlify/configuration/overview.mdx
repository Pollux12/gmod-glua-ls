## Overview

All language server behavior is configured through a `.gluarc.json` file placed in your workspace root. The file has full JSON schema support, so your editor will validate settings and provide completions as you type.

---

## Creating your config

Create a `.gluarc.json` file at the root of your workspace:

```json .gluarc.json
{
  "$schema": "https://raw.githubusercontent.com/Pollux12/gmod-glua-ls/refs/heads/main/crates/glua_code_analysis/resources/schema.json"
}
```

The `$schema` property enables IDE validation and autocomplete for the file itself.

Alternatively, the VSCode extension can generate a starter configuration for you — open the command palette and run **GLua: Open Settings UI**, then click the **Edit in JSON** button at the bottom of the panel.

---

## Recommended starter config

This configuration is suitable for most GMod addon projects:

```json .gluarc.json
{
  "$schema": "https://raw.githubusercontent.com/Pollux12/gmod-glua-ls/refs/heads/main/crates/glua_code_analysis/resources/schema.json",
  "diagnostics": {
    "enable": true,
    "diagnosticInterval": 500,
    "severity": {
      "unused": "hint",
      "undefined-field": "information",
      "redundant-return": "hint",
      "redundant-return-value": "hint",
      "param-type-mismatch": "information",
      "missing-fields": "information",
      "assign-type-mismatch": "information",
      "return-type-mismatch": "information",
      "missing-parameter": "information",
      "cast-type-mismatch": "information",
      "need-check-nil": "hint"
    }
  }
}
```

## Config file priority

When multiple config files are present, GLuaLS follows this priority order (highest first):

1. `.gluarc.json`
2. `.luarc.json`
3. `.emmyrc.json`
4. `.emmyrc.lua`

`.gluarc.json` is exclusive — if present, all other config files are ignored.

---

## VS Code settings

Some behaviors can be configured directly in VS Code settings (`settings.json` or the **Settings UI**) under the `gluals.*` prefix. These include:

| Setting | Description |
|---|---|
| `gluals.gmod.autoLoadAnnotations` | Auto-download and load GMod annotations |
| `gluals.gmod.annotationsPath` | Custom path to annotations to load |
| `gluals.debugger.stopOnEntry` | Break on first line when debugger attaches |
| `gluals.debugger.stopOnError` | Pause on Lua errors during debug |

Settings in `.gluarc.json` always take precedence over VS Code workspace settings for analysis behavior.
