# Contribute to the documentation

Thank you for your interest in contributing to our documentation! This guide will help you get started.

## How to contribute

### Option 1: Edit directly on GitHub

1. Navigate to the page you want to edit
2. Click the "Edit this file" button (the pencil icon)
3. Make your changes and submit a pull request

### Option 2: Local development

1. Fork and clone this repository
2. Install the Mintlify CLI: `npm i -g mint`
3. Create a branch for your changes
4. Make changes
5. Navigate to the docs directory and run `mint dev`
6. Preview your changes at `http://localhost:3000`
7. Commit your changes and submit a pull request

For more details on local development, see our [development guide](development.mdx).

## Writing guidelines

Write so a new GMod developer can follow the page, while still being clear and useful for experienced developers.

- Use plain words first. Prefer "use" over "utilize", "help" over "facilitate", and "set up" over "configure" when both are accurate.
- Keep sentences short. One idea per sentence is the default.
- Lead with the action. Start sections with what the reader should do.
- Explain terms once. If a term is advanced, define it briefly the first time it appears.
- Do not overexplain. Include only what helps the reader complete the task.
- Use consistent terms across pages for the same concept.
- Prefer practical examples. Show the annotation or config that readers should copy and adapt.
- Keep warnings direct and specific. State what breaks and what to do instead.
