## Multi-root workspaces <a id="multi-root-workspaces" />

If you work on a gamemode and multiple addons, use a [multi-root workspace](https://code.visualstudio.com/docs/editing/workspaces/multi-root-workspaces) and add each as its own folder.

By default, multi-root workspace isolation is disabled. The language server merges workspace configs into one global baseline (first workspace config wins scalar conflicts, arrays are unioned), while still allowing per-workspace file-scoped overrides where needed.

You can enable isolation with `workspace.enableIsolation` when you want each folder analyzed separately. See [Configuration](/configuration/workspace) for details.

<Note>
Do not open the entire `garrysmod/` folder as your workspace root. GLuaLS expects each root to be an individual addon or gamemode, not the entire game directory.
</Note>

## Library paths <a id="library-paths" />

If your project uses shared utility files outside the workspace root, add them as library paths:

<ConfigTip />
```json .gluarc.json
{
  "workspace": {
    "library": [
      "/path/to/shared-utils",
      {
        "path": "./external/mylibrary",
        "ignoreDir": ["tests"],
        "ignoreGlobs": ["**/*.spec.lua"]
      }
    ]
  }
}
```

Library files are analyzed, and their types are available in your workspace. Diagnostics are not reported for library files.

## Ignoring files and folders <a id="ignoring-files-and-folders" />

<ConfigTip />

```json .gluarc.json
{
  "workspace": {
    "ignoreDir": ["build", "dist", ".git"],
    "ignoreGlobs": ["**/*.min.lua", "tests/**"]
  }
}
```

## Custom require patterns <a id="custom-require-patterns" />

If your project uses a custom module loader, map it to `require` behavior:

<ConfigTip />

```json .gluarc.json
{
  "runtime": {
    "requireLikeFunction": ["include", "MYADDON.Require"],
    "requirePattern": ["?.lua", "?/init.lua"]
  }
}
```

## Annotations path <a id="annotations-path" />

By default, the VS Code extension downloads and updates GMod wiki annotations. If you need a custom annotations build or want auto-download off:

<ConfigTip />

```json .gluarc.json
{
  "gmod": {
    "annotationsPath": "./my-custom-annotations",
    "autoLoadAnnotations": false
  }
}
```

See [Annotations management](/configuration/annotations-management) for details on how auto-download works.

## Framework plugin quick start

If you are using a framework (for example DarkRP or Helix):

1. Open your project folder in VS Code.
2. Wait for the plugin detection prompt.
3. Click **Apply** to apply plugin settings to `.gluarc.json`.

If no prompt appears, open Command Palette and run:

- **GLuaLS: Framework/Plugin Setup Wizard**
- **GLuaLS: Re-run Framework/Plugin Detection**
- **GLuaLS: Apply Framework Plugin** (manual override)

Use the setup wizard first. Use the other two commands only if needed.

For full plugin usage and custom plugin creation, see [Framework plugins](/configuration/framework-plugins).
